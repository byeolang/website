# byeol 홈페이지 ScrollyTelling 리소스 정리

## 목적
이 문서는 메인 홈페이지의 4개 Scene에 필요한 리소스를 정리하고, 각 리소스를 어떤 방식으로 표현할지(2D / 2.5D / 3D), 어떤 기술이 담당할지(GSAP / ScrollTrigger / Three.js)를 한 번에 보기 위해 작성한다.

---

## 전체 표현 원칙

- 기준 비율: **16:9**
- 아트 스타일: **시네마틱 픽셀아트**
- 텍스트 / 설명 / 코드카드 / 차트카드 / 버튼: **이미지로 박지 않고 UI로 구현**
- 텍스트 뒤에는 **검정 → 투명 그라데이션 백플레이트** 사용
- 카메라 연출은 영화처럼 역동적으로 가져가되, Scene 전환은 자연스럽게 연결
- Scene 1 시작 프레임은 **현재 홈페이지 대문 이미지 그대로** 사용

---

## 기술 역할 분담

### GSAP + ScrollTrigger
- 전체 스크롤 진행 제어
- Scene 전환 타이밍 제어
- DOM/UI 카드 등장·퇴장
- 이미지 위치, 크기, 투명도, 회전 변화
- 텍스트, 코드카드, 플로우카드, 오류카드, CTA 버튼 애니메이션

### Three.js
- 깊이감과 공간감 표현
- 3D 천체, 소행성, 별/먼지 파티클
- 카메라 fly-through 느낌 보강
- Scene 3 중심의 입체적 우주 연출
- Scene 4의 행성/위성/별자리 depth 보강

### 이미지 리소스
- 픽셀아트 배경
- 로켓 본체
- 발사대
- 구름, 성운, 지면
- 불꽃, 연기, 먼지
- 소행성, 별, 별자리 라인용 베이스 자산

---

## 표현 방식 요약

| 대상 | 표현 방식 | 주 담당 |
|---|---|---|
| 현재 대문 배경 | 2D | 이미지 + GSAP |
| 로켓 | 2.5D 스프라이트 (Scene별 구도 교체) | 이미지 + GSAP |
| 발사대 | 2D | 이미지 + GSAP |
| 구름층 | 2D 레이어 / 얕은 2.5D | 이미지 + GSAP, 일부 Three.js 가능 |
| 불꽃 / 연기 / 먼지 | 2D 파티클성 PNG | 이미지 + GSAP |
| 지구 / 행성 / 위성 | 3D sphere 권장 | Three.js |
| 소행성 | 3D 메시 권장 | Three.js |
| 별 / 우주먼지 / 파티클 | 3D 또는 2.5D 파티클 | Three.js |
| 성운 | 2D 배경 + 약한 패럴랙스 | 이미지 + GSAP |
| 별자리 라인 | 2D 또는 3D line | GSAP 또는 Three.js |
| 코드 / 차트 / 오류카드 / 버튼 | UI | DOM + GSAP |

# Scene별 리소스 정리

## Scene 1 — 점화 + 이륙
### 기능
**적은 문법, 빠른 이해**

### 장면 역할
현재 대문에서 바로 시작해, 왼쪽 언덕 위 발사대에서 로켓이 점화되고 이륙한다.
중앙 로고/타이틀/BEGIN QUEST는 스크롤 시작과 함께 fade out 된다.

### 필요한 리소스

#### 배경 / 환경
- 현재 홈페이지 대문 배경
- 왼쪽 언덕 위 발사대

#### 주인공
- 로켓 측면 스프라이트

#### 이펙트
- 점화 불꽃 2~3단계
- 지상 연기
- 발사 먼지
- 작은 불티/파티클

#### UI
- 제목/설명용 그라데이션 백플레이트
- 키워드 수 강조 배지
- C# vs byeol 코드 비교 카드

### 표현 방식
| 리소스 | 표현 방식 | 기술 |
|---|---|---|
| 현재 대문 배경 | 2D | 이미지 |
| 발사대 | 2D | 이미지 |
| 로켓 | 2.5D 스프라이트 | 이미지 + GSAP |
| 점화 불꽃 | 2D | 이미지 + GSAP |
| 연기 / 먼지 | 2D | 이미지 + GSAP |
| 코드 비교 카드 | UI | DOM + GSAP |
| 키워드 수 배지 | UI | DOM + GSAP |

### 기술 사용 메모
- **Three.js는 거의 사용하지 않음**
- Scene 1은 현재 대문 이미지와의 합성이 핵심이므로 **이미지 + GSAP 중심**으로 처리

---

## Scene 2 — 구름층 통과 + 대기권 돌파
### 기능
**빌드 벽 없는 개발**

### 장면 역할
로켓이 구름층을 뚫고 상승하고, 대기권을 돌파한다.
빌드 대기 없는 빠른 개발 루프를 추진력과 상승으로 상징한다.

### 필요한 리소스

#### 배경 / 환경
- 구름 원경 레이어
- 구름 중경 레이어
- 구름 전경 레이어
- 상층 대기 배경
- 대기권 경계 빛 링 / 충격파 효과

#### 주인공
- 상승용 로켓 스프라이트

#### 이펙트
- 강한 엔진 불꽃
- 속도선 / 잔광
- 구름 사이 안개 / 미스트

#### UI
- 제목/설명용 그라데이션 백플레이트
- 일반 개발 플로우 vs byeol 플로우 비교 카드

### 표현 방식
| 리소스 | 표현 방식 | 기술 |
|---|---|---|
| 구름 레이어 | 2D 레이어 / 얕은 2.5D | 이미지 + GSAP, 일부 Three.js 가능 |
| 상층 대기 배경 | 2D | 이미지 |
| 대기권 돌파 효과 | 2D 이펙트 | 이미지 + GSAP |
| 로켓 | 2.5D 스프라이트 | 이미지 + GSAP |
| 불꽃 / 속도선 | 2D | 이미지 + GSAP |
| 플로우 비교 카드 | UI | DOM + GSAP |

### 기술 사용 메모
- 기본은 **이미지 + GSAP 중심**
- 필요하면 구름 plane만 z축으로 분리해서 **얕은 Three.js depth**를 줄 수 있음
- 풀 3D보다는 **2.5D 하이브리드**가 적합

---

## Scene 3 — 우주 항해 + 소행성 지대
### 기능
**스크립트처럼 가볍게, 오류는 먼저**

### 장면 역할
지구를 떠난 뒤 우주를 항해하고, 소행성 지대를 지난다.
가볍고 민첩하게 움직이되, 위험은 미리 탐지하는 항법 감각을 보여준다.

### 필요한 리소스

#### 배경 / 환경
- 깊은 우주 배경
- 하단에 살짝 보이는 지구
- 성운 배경
- 원경 별빛
- 중경 별빛

#### 주인공
- 후방-사선 구도의 로켓 스프라이트

#### 장애물 / 공간 오브젝트
- 대형 소행성 2종 이상
- 중형 소행성 2종 이상
- 소형 소행성 클러스터
- 카메라 전경을 스치는 블러 소행성

#### 이펙트
- 긴 우주용 엔진 불꽃
- 속도선
- 우주 먼지 / 파티클
- 경고 마커 / 탐지 효과

#### UI
- 제목/설명용 그라데이션 백플레이트
- 짧은 코드 카드
- 정적 오류 탐지 카드

### 표현 방식
| 리소스 | 표현 방식 | 기술 |
|---|---|---|
| 우주 배경 / 성운 | 2D + 약한 패럴랙스 | 이미지 + GSAP |
| 지구 | 3D sphere 권장 | Three.js |
| 로켓 | 2.5D 스프라이트 | 이미지 + GSAP |
| 소행성 | 3D 메시 권장 | Three.js |
| 별 / 우주먼지 / 파티클 | 3D 또는 2.5D 파티클 | Three.js |
| 코드 카드 | UI | DOM + GSAP |
| 오류 탐지 카드 | UI | DOM + GSAP |
| 경고 마커 | UI 또는 2D 이펙트 | DOM + GSAP / Three.js 보조 |

### 기술 사용 메모
- **Three.js 활용의 핵심 Scene**
- 지구, 소행성, 우주 파티클, 카메라 전경 스침 효과는 Three.js가 담당
- GSAP는 텍스트/UI와 진행 타이밍 동기화를 담당

---

## Scene 4 — 행성 발견 + 착륙
### 기능
**AI 이후에도, 계속 만들고 싶은 언어**

### 장면 역할
새로운 행성을 발견하고 착륙한다.
이 장면은 사용자가 코드로 완성한 프로덕트 세계에 도착하는 순간이며, 마지막에는 메시지와 다운로드 CTA가 등장한다.

### 필요한 리소스

#### 배경 / 환경
- 행성 표면 배경
- 행성 지평선 광원
- 최종 우주 하늘 배경
- 성운 배경
- 큰 행성 또는 위성
- 별자리 라인 효과

#### 주인공
- 착륙한 로켓 스프라이트

#### 이펙트
- 착륙 먼지
- 약한 엔진 잔광
- 별빛 / 미세 파티클

#### UI
- 제목/설명용 그라데이션 백플레이트
- 다운로드 CTA 버튼

### 표현 방식
| 리소스 | 표현 방식 | 기술 |
|---|---|---|
| 행성 표면 / 하늘 / 성운 | 2D | 이미지 + GSAP |
| 로켓 | 2.5D 스프라이트 | 이미지 + GSAP |
| 착륙 먼지 | 2D | 이미지 + GSAP |
| 큰 행성 / 위성 | 3D sphere 권장 | Three.js |
| 별자리 라인 | 2D 또는 3D line | GSAP 또는 Three.js |
| 다운로드 버튼 | UI | DOM + GSAP |

### 기술 사용 메모
- Scene 4는 **이미지 + GSAP 중심**, Three.js는 분위기 보강용으로만 사용
- 핵심은 3D 과시가 아니라 **도착감, 감성, CTA**

# Scene별 기술 우선도

| Scene | GSAP 비중 | Three.js 비중 | 핵심 방식 |
|---|---:|---:|---|
| Scene 1 | 매우 높음 | 매우 낮음 | 2D/2.5D |
| Scene 2 | 높음 | 낮음~중간 | 2.5D 하이브리드 |
| Scene 3 | 높음 | 매우 높음 | 2.5D + 3D 하이브리드 |
| Scene 4 | 높음 | 중간 | 2.5D + 분위기용 3D |

# 최종 구현 방향 한 줄 요약

- **로켓은 끝까지 2.5D 스프라이트 중심**으로 간다.
- **지구, 행성, 위성, 소행성은 3D가 효과적**이므로 Three.js를 쓴다.
- **텍스트, 차트, 코드, 오류카드, 버튼은 DOM UI + GSAP**로 처리한다.
- **Scene 3에서 Three.js를 가장 강하게 사용**하고, Scene 1·2·4는 보조적으로만 사용한다.
